# 리뷰 가이드

## 목적
- 설계의 경계/책임/트랜잭션을 검증하고, 토론 가능한 질문을 남긴다.

## 절차
1. PR 템플릿의 요구사항/설계 요약을 읽어 컨텍스트 파악.
2. `review_questions.md`를 참고하여 논의가 필요한 영역을 질문 위주로 피드백.
3. 정답 제시는 피하고, 대안/리스크 질문을 남긴다.
4. 최소 2개의 질문형 피드백 후 승인/요청변경 여부 결정.

## 체크리스트
- [ ] 요구사항과 DDD 분석이 연결되는지 확인했다.
- [ ] 시나리오와 테스트가 일관된지 검토했다.
- [ ] AI 질문으로 변경된 부분이 실제 반영되었는지 살폈다.
- [ ] 오프라인 토론에 가져갈 쟁점을 작성했다.

## 팁
- "이것이 맞다" 대신 "경계를 이렇게 나눴을 때 어떤 리스크가 보이나요?"처럼 질문형 코멘트를 남긴다.
- 승인 여부가 명확하지 않으면 `needs-discussion` 레이블을 붙여 오프라인에서 다룬다.
