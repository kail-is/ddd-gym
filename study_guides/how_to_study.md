# 어떻게 학습할까

AI START 챗봇이 템플릿 복사, 폴더 생성, 체크리스트 갱신을 책임지고 안내하므로 참가자는 질문에 답하고 \"반영했다\"고 확인만 하면 됩니다. 파일을 직접 찾아 편집할 일은 재진입이나 수동 수정이 필요할 때뿐입니다.

## 재진입 가이드
"지금 어디까지 했는지 모르겠다면 여기서 시작":
1. `participants/<이름>/` 폴더의 최신 PR을 확인합니다.
2. 마지막으로 수정한 문서 상단의 STEP과 체크리스트를 읽고 완료 여부를 표시합니다.
3. 체크리스트가 일부만 완료라면, 남은 항목만 이어서 작성합니다.
4. START 챗봇이 재시작 질문을 할 수 있도록 `participants/<이름>/step0_prepare/weekly_progress.md`에서 완료된 주차를 체크하고 다음 액션을 적어 둡니다.

## 최소 제출 기준
- 요구사항 템플릿의 모든 항목을 개략적으로라도 채운다.
- DDD 분석 템플릿에서 컨텍스트/애그리게잇 후보를 최소 1개씩 제시한다.
- 시나리오와 테스트는 정상 케이스 1개씩만 있어도 제출 가능하다.
- 질문 루프는 질문 3개만 기록해도 된다.

## 권장 루틴
1. **집중 25분**: 요구사항 정리 → 분석 초안 작성.
2. **5분 휴식 후**: 시나리오/테스트 케이스 초안 작성.
3. **집중 25분**: AI 질문 루프 돌리고 설계 수정.
4. **마감 10분**: PR 템플릿 채우고, 다음 세션 토론에서 다룰 포인트 정리.

## 체크리스트
- [ ] 요구사항-분석-시나리오-테스트-질문 루프 문서를 모두 찾았다.
- [ ] 각 문서의 STEP과 체크리스트를 확인했다.
- [ ] 최소 제출 기준을 이해했다.
- [ ] 다음에 작성할 단계가 명확하다.
- [ ] START 챗봇을 위한 `participants/<이름>/step0_prepare/weekly_progress.md` 로그를 업데이트했다.

## 시간 관리 팁
- 하루 30분이라도 작성하여 PR을 올리면, 다음 토론에서 바로 이어갈 수 있습니다.
- 문서는 독립이므로 요구사항만 작성한 PR도 허용되며, 토론 파트너가 다음 질문을 남길 수 있도록 템플릿을 그대로 제출하세요.
