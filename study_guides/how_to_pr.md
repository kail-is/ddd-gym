# PR 작성 방법

## 목적
- 토론 가능한 중간 산출물을 공유하여 빠르게 피드백을 받는다.

## 절차
1. 브랜치명: `feature/<주차>-<주제>`.
2. `participants/<이름>/stepX_*` 폴더에 템플릿 기반 문서 추가.
3. `.github/pull_request_template.md`에 맞춰 PR 작성.
4. PR 설명에 오프라인 토론 후보 여부와 참고 링크를 명시한다.

## 체크리스트
- [ ] 요구사항~질문 루프 문서 링크 첨부.
- [ ] 핵심 결정 2개와 버린 대안 1개 이상 작성.
- [ ] 시나리오/테스트 결과와 AI 질문 반영 여부 명시.
- [ ] 불완전한 부분이 있다면 TODO로 표시.

## 팁
- 주석 대신 PR 설명에서 의도와 미완료 항목을 적는다.
- 토론이 필요한 부분은 `> 논의필요:` 블록을 활용해 강조한다.
