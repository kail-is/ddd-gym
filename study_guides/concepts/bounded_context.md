# 개념-바운디드 컨텍스트

## 한 줄 정의
하나의 언어와 모델이 일관되게 동작하도록 경계 지어진 설계 공간.

## 판단 기준
- 용어가 다른 의미로 해석되는 순간이다.
- 데이터 일관성 요구(강/약)를 명확히 구분할 필요가 있다.
- 팀/시스템 책임이 분리되어야 한다.

## 안티패턴
- UI/배포 단위와 동일시하여 개념적 경계가 흐려짐.
- 컨텍스트 내부에서 다시 별도 언어를 섞어 혼합 모델을 만든다.

## 스터디 적용 위치
- `ddd_analysis.md`의 컨텍스트 후보.
- 오프라인 토론에서 PR 비교 시 경계 조합 논의.
