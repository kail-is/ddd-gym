- [x] 정상/예외/경계/위험 케이스를 각각 최소 1개 작성했다.

## 예약 상태 전이 테스트

| 분류 | 케이스명 | 조건 | 기대 결과 | 참고 시나리오 |
| --- | --- | --- | --- | --- |
| 정상 | 예약 요청 → 확정 | 가용일 존재, 잠금 없음, 결제 성공 | 상태 `확정됨`, `BookingConfirmed` 발행 | 예약 요청 → 결제 → 확정 |
| 정상 | 확정 → 체크인 → 체크아웃 | 확정됨 상태, 체크인 날짜 도래 | `체크인` → `체크아웃` 순서대로 전이 | 체크인 → 체크아웃 |
| 정상 | 취소 요청 → 호스트 확정 → 환불 | 확정됨 상태, 고객 취소 요청, 호스트 확정 | CancellationRequest 생성 → CancellationApproved → cancel() → 환불 | 고객 취소 요청 → 호스트 확정 → 환불 |
| 예외 | 취소된 예약 재확정 | 취소됨 상태에서 confirm() 호출 | `require` 실패, 상태 변경 없음 | 취소된 예약 재확정 시도 |
| 예외 | 체크인 없이 체크아웃 | 확정됨 상태에서 checkOut() 호출 | `require` 실패, 상태 변경 없음 | 체크인 없이 체크아웃 시도 |
| 예외 | 체크인 후 취소 시도 | 체크인 상태에서 cancel() 호출 | `require` 실패, 상태 변경 없음 | (불변조건 검증) |

## 이선좌 잠금 테스트

| 분류 | 케이스명 | 조건 | 기대 결과 | 참고 시나리오 |
| --- | --- | --- | --- | --- |
| 정상 | 잠금 중 재시도 성공 | 폴링 유지, 결제 재시도 | 확정 처리 | 결제 실패 후 재시도 성공 |
| 예외 | 잠금 중 다른 사용자 예약 | 타 사용자가 동일 날짜 요청 | 예약 거절 | 이선좌 잠금 중 다른 사용자 예약 시도 |
| 경계 | 폴링 끊김 직후 (10분 미만) | 마지막 heartbeat 후 9분 59초 | 잠금 유지, 재시도 가능 | (경계 검증) |
| 경계 | 폴링 끊김 + 10분 정확히 경과 | 마지막 heartbeat 후 10분 | `BookingExpired` 발행, 잠금 해제, 가용일 복구 | 결제 실패 + 폴링 끊김 → 잠금 해제 |
| 위험 | 잠금 해제 직전 결제 성공 | 9분 59초에 결제 성공 요청 | 잠금 유지 중이므로 확정 처리. 단, 해제 스케줄러와 경합 가능 | (경합 시나리오) |
| 위험 | 잠금 해제와 결제 성공 동시 | 10분 경과 시점에 결제 성공과 잠금 해제가 동시 발생 | 하나만 성공해야 함 (원자적 처리 필요) | (경합 시나리오) |

## 취소 요청 (CancellationRequest) 테스트

| 분류 | 케이스명 | 조건 | 기대 결과 | 참고 시나리오 |
| --- | --- | --- | --- | --- |
| 정상 | 취소 요청 생성 | 확정됨 상태, 고객 취소 요청 | CancellationRequest 생성, Booking 상태 변경 없음 (`확정됨` 유지) | 고객 취소 요청 → 호스트 확정 → 환불 |
| 정상 | 호스트 취소 확정 | CancellationRequest 대기 중, 호스트 확정 | CancellationApproved 발행 → Booking.cancel() → BookingCancelled 발행 | 고객 취소 요청 → 호스트 확정 → 환불 |
| 정상 | 호스트 취소 거절 | CancellationRequest 대기 중, 호스트 거절 | CancellationRejected 발행, Booking 영향 없음, 체크인 가능해짐 | 호스트가 취소 거절 |
| 예외 | 취소 요청 대기 중 체크인 시도 | CancellationRequest 대기 중, 고객 체크인 시도 | 서비스에서 큐 확인 후 체크인 거절 | 취소 요청 대기 중 체크인 시도 |
| 경계 | 취소 거절 직후 체크인 | 호스트 거절 → 즉시 체크인 시도 | CancellationRequest 처리 완료 상태이므로 체크인 허용 | (경계 검증) |
| 위험 | 취소 확정과 체크인 동시 | 호스트 취소 확정과 고객 체크인이 동시 발생 | 하나만 성공해야 함 (서비스 레벨에서 원자적 처리 필요) | (경합 시나리오) |

## 환불 계산 테스트

| 분류 | 케이스명 | 조건 | 기대 결과 | 참고 시나리오 |
| --- | --- | --- | --- | --- |
| 정상 | 전액 환불 (거절) | 호스트가 예약 거절 | 전액 환불 요청 | (Week1 플로우) |
| 정상 | 조건부 환불 (취소) | 확정 후 고객 취소, 체크인 전 | 예약 컨텍스트가 계산한 금액으로 환불 | 확정 후 취소 → 환불 |
| 경계 | 환불 금액 = 결제 금액 | 전액에 해당하는 조건부 환불 | Payment 상태가 REFUNDED (PARTIALLY_REFUNDED 아님) | (경계 검증) |

## (학습 목적) 부분 취소 테스트

| 분류 | 케이스명 | 조건 | 기대 결과 | 참고 시나리오 |
| --- | --- | --- | --- | --- |
| 정상 | 1박 부분 취소 | 3박 중 1박 취소 요청 | 해당 BookingItem 취소, Booking은 확정됨 유지, 1박 분 환불 | 3박 중 1박 부분 취소 |
