# 시나리오 템플릿 (Given/When/Then)

## 체크리스트
- [x] 정상 시나리오 1개 이상.
- [x] 예외 시나리오 1개 이상.
- [x] 각 시나리오는 Given/When/Then 구조를 따른다.
- [x] 상태 변화가 명확히 드러난다.

## 포맷

### 1. 체크인 전 확정 – 정상
- **종류**: 정상
- **Given**: 숙소 A의 8/1~8/3 재고가 비어 있고, 호스트는 정산 보류 상태가 아니다.
- **When**: 고객 B가 해당 기간으로 예약을 요청한다.
- **Then**: 예약 상태가 `확정됨`으로 전환되고 `BookingConfirmed` 이벤트가 발행된다.

### 2. 정산 보류 거절 – 예외
- **종류**: 예외
- **Given**: 호스트 C는 정산 컨텍스트에서 `보류`로 표시되어 있다.
- **When**: 고객이 숙소 C에 예약을 요청한다.
- **Then**: 예약은 `거절됨` 처리되고 사유 "정산 보류"가 Booking 애그리게잇에 기록된다.

### 3. 체크인 이후 취소 시도 – 경계
- **종류**: 예외
- **Given**: 예약 D는 이미 `체크인` 상태다.
- **When**: 고객이 취소 버튼을 누른다.
- **Then**: 취소 요청이 거절되고 "체크인 이후 취소 불가" 사유를 기록하며 정산 컨텍스트에 아무 이벤트도 발행하지 않는다.

### 4. 체크아웃 순서 검증 – 위험
- **종류**: 위험
- **Given**: 예약 E는 `확정됨` 상태지만 아직 `체크인`되지 않았다.
- **When**: 파트너 시스템이 실수로 `체크아웃` API를 호출한다.
- **Then**: StayProgress 애그리게잇이 요청을 거절하고 감사 로그를 남겨서 상태 역행을 방지한다.
