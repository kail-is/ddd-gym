# Week 2 – Aggregate/Event/Scenario/Test

## 공통 목표
- 애그리게잇 책임 정의.
- Domain Event 목록 확정.
- 시나리오와 테스트 초안 작성.

## 실험 미션
- Week1에서 만든 기본 설계를 **의도적으로 깨뜨린 뒤** 다시 세웁니다.
- 예약 한 건을 택해 \"취소/체크인/체크아웃\" 플로우의 약점을 찾아보고, 질문 루프와 함께 재설계합니다.
- 설계-공유-토의 구조를 유지하기 위해 각 변경 이유와 영향을 `ddd_analysis.md` 트레이드오프 섹션에 기록하세요.

## 제약 리마인더
- 락/분산락 없이도 책임과 경계로 해결하길 권장합니다.
- 규칙이 흩어지는 순간을 시나리오/테스트로 바로 검증하세요.

## 토론 질문
- 애그리게잇 루트를 어디까지 노출했나?
- 이벤트를 통해 어떤 정책을 decouple 했나?
- 시나리오와 테스트가 동일한 상태 전이를 검증하는가?
