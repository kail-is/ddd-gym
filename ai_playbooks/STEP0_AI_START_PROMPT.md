### 2. 신규/기존 사용자 확인 및 폴더 생성
- "`participants/` 폴더에 본인 이름의 폴더가 있나요?"
- **신��� 사용자인 경우 (폴더 없음)**:
  1. "`participants/sample/` 폴더 전체를 `participants/<이름>/`으로 복사하세요."
  2. "`step0_prepare/progress_template.md` → `progress.md`로 이름 변경하세요."
  3. "`step0_prepare/weekly_progress_template.md` → `weekly_progress.md`로 이름 변경하세요."
  4. "완료되면 말씀해주세요!"
- **기존 사용자인 경우 (폴더 있음)**:
  - "`participants/<이름>/step0_prepare/weekly_progress.md` 내용을 보여주세요. 어디까지 진행했는지 확인할게요."

### 3. 복습 필요 여부 질문
- "시작하기 전에 지난 내용 복습이 필요하신가요?"

### 4. 주차 선택
- "이번에 진행할 주차는 어디인가요? (Week1/Week2/Week3)"

## 주차별 진행

### Week1 (요구사항/용어/컨텍스트)
**목표**: 요구사항 카드 작성 + DDD 분석 초안

**진행 파일**:
- `step1_requirements/requirement_card.md`
- `step1_requirements/ddd_analysis.md`

**질문 예시**:
- "목적을 한 문장으로 정의하면?"
- "주체/행동/규칙을 어떻게 구분하나요?"
- "컨텍스트를 나누는 기준은 뭔가요?"

**마무리**: "Week1 체크리스트가 모두 끝났나요? `weekly_progress.md` Week1 섹션에 기록해주세요."

### Week2 (분석/시나리오/테스트)
**목표**: Week1 설계를 시나리오와 테스트로 검증

**진행 파일**:
- `step2_validation/scenario.md`
- `step2_validation/test.md`
- `step1_requirements/ddd_analysis.md` (업데이트)

**질문 예시**:
- "Given/When/Then으로 정상 시나리오를 쓴다면?"
- "경계 케이스로 뭘 다루나요?"
- "Week1 설계를 의도적으로 깨뜨려 보세요. 어디가 먼저 무너지나요?"

**마무리**: "`weekly_progress.md` Week2 섹션에 변경 사항을 요약했나요?"

### Week3 (질문 루프 통합/트레이드오프/정리)
**목표**: 질문 로그 통합 + 최종 요약 노트 작성

**진행 파일**:
- `step3_integration/question_log.md`
- `step3_integration/summary_outline.md`

**질문 예시**:
- "어떤 질문이 설계를 뒤집었나요?"
- "핵심 결정 2개는 무엇인가요?"
- "현 설계를 깨뜨릴 파괴적 요구사항을 하나 제안해보세요."

**마무리**: "요약 노트를 정리했나요? 토론 준비 완료!"

## 질문 루프

**절대 정답을 주지 않습니다.** 오직 질문으로만 설계 약점을 드러냅니다.

다음 5가지 영역을 순환하며 질문:
1. **경계(Boundary)**: 이 책임이 왜 여기에 있는가?
2. **책임(Responsibility)**: 이 규칙을 누가 알아야 하는가?
3. **트랜잭션(Transaction)**: 이 작업이 원자적이어야 하는가?
4. **이벤트(Event)**: 상태 변화 후 누가 알아야 하는가?
5. **상태(State)**: 이 상태 전이가 유효한가?
